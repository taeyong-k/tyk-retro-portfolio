## 🔥트러블 슈팅

### 1. LocomotiveScroll + 툴바 이동 스크롤 튐 문제

- **문제점**
    - 툴바 버튼 클릭 후 섹션 이동 직후, 스크롤하면 이전 위치로 순간이동.
    - 원인: LocomotiveScroll의 smooth 보간값과 브라우저 native scroll 위치가 불일치.
- **해결**
    - ScrollSmoother로 교체, `scrollTop`과 내부 스무딩 값을 JS로 동기화.
    - 버튼 클릭 시 `scrollSmoother.scrollTo(target, {duration: 0.5, offset: 0})`로 위치 강제 설정.
    - 이동 중 다른 scroll 이벤트를 무시하도록 상태 플래그(`isScrolling`) 추가.
- **회고**
    - smooth scroll 엔진의 내부 좌표와 DOM scroll 위치 동기화 중요성 확인.
    - 외부 라이브러리 적용 시 edge case를 예상하고 이벤트 플래그로 제어 필요.

---

### 2. 모달 닫을 때 애니메이션 재실행 문제

- **문제점**
    - 모달 재오픈 시 애니메이션 중복 실행 및 scroll 위치 초기화.
    - 원인: ScrollSmoother의 스크롤 상태와 모달 이벤트 충돌.
- **해결**
    - 모달 열기 전 `scrollSmoother.paused(true)`, 닫기 후 `scrollSmoother.paused(false)`로 잠금/해제.
    - 애니메이션 트리거를 `once: true` → 재실행 시 상태 초기화(`animation.restart()`)로 변경.
- **회고**
    - UI 이벤트와 스크롤 상태 동기화 없이는 반복 컴포넌트 안정적 동작 어려움.
    - 상태 플래그와 라이프사이클 관리가 필수적임.

---

### 3. LED 보드 반응형 문제

- **문제점**
    - 화면 축소 시 dot 크기와 텍스트 비율 깨짐, CSS만으로 유지 어려움.
- **해결**
    - CSS `aspect-ratio`로 dot 가로세로 비율 유지, JS로 동적으로 컬럼 수 계산(`Math.floor(containerWidth / dotSize)`), dot position과 text 위치 재계산.
    - resize 이벤트마다 재렌더링하여 레이아웃 깨짐 방지.
- **회고**
    - 동적 레이아웃은 CSS + JS 협업 필요.
    - 단순 CSS는 비율 유지 불가, JS로 container width 기반 재계산 필수.

---

### 4. 커서 위치 문제 (중앙 정렬)

- **문제점**
    - 커서가 화면 중앙 기준으로 정확히 위치하지 않음.
    - 원인: pointermove 이벤트 기준 좌표와 transform-origin 불일치.
- **해결**
    - JS에서 커서 좌표(`clientX/Y`) 가져와 `transform: translate(-50%, -50%)` 적용.
    - 이벤트 초기화 시점 DOM 크기 계산 후 중앙 기준 재정렬.
    - window resize 시 `updateCursorPosition()` 호출해 위치 유지.
- **회고**
    - UI 디테일 조정은 좌표 기준과 DOM 렌더링 시점 이해 필수.
    - 작은 오차도 UX에 큰 영향.

---

### 5. 오프닝 타이핑/애니메이션 색상 충돌

- **문제점**
    - 타이핑 애니메이션과 배경색 겹침으로 글자 가시성 저하.
    - z-index와 애니메이션 속성 충돌.
- **해결**
    - 애니메이션 레이어 구조 재설계: `typingLayer`와 `backgroundLayer` 분리.
    - z-index: `typingLayer: 10`, `backgroundLayer: 1`로 명확히 지정.
    - GSAP timeline에 `opacity`, `color` 속성만 적용하고, 불필요한 transform/scale 속성 제거.
    - 애니메이션 순서 제어: background fade-in → typing 시작 → 글자 색상 transition.
- **회고**
    - 레이어 구조와 애니메이션 속성 충돌 최소화 필요.
    - 초기 설계 단계에서 구조적 분리와 timeline 순서 제어가 안정적 UI 핵심.

---

### 6. 갤러리 애니메이션 위치 어긋남 문제

- **문제점**
    - 갤러리 애니메이션 실행 중, 크롬 브라우저에서 이미지가 원래 위치보다 아래로 어긋나는 현상이 발생함.
    - 엣지에서는 동일 환경에서도 문제 없이 정상 작동.
    - 원인: `runAnimation()`에서 모든 이미지의 `x`와 `y` 좌표를 0으로 초기화하면서, 원형 배치용 `transformOrigin` 기준 좌표가 변해 시각적으로 이미지가 아래로 내려오는 현상이 발생함.
- **해결**
    - runAnimation과 resetAnimation에서 `x`와 `y` 초기화를 제거하고, `rotation`만 초기화하도록 변경.
    - 이렇게 하면 transformOrigin 기준으로 원형 배치가 유지되어 위치 튐 현상이 사라짐.
- **회고**
    - 초기 애니메이션과 리셋 시 불필요한 좌표 초기화는 `transform` 구조를 깨뜨려 브라우저 간 동작 차이를 유발할 수 있음.
    - 최소한의 속성만 초기화하도록 설계하면 모든 브라우저에서 안정적으로 작동함을 확인.
